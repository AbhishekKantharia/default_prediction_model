Problem Statement 4

Loan Default Prediction Model

A robust ML solution that estimates default probability 12 months in advance, using both structured and unstructured data, achieving 90%+ accuracy across all loan types and borrower segments.

12-Month Horizon 90% Target Accuracy Structured + Unstructured
The Challenge

Current State vs Expected Outcome

Current Limitations

  • Low prediction accuracy (16–22%)
  • Dependent solely on structured data
  • Fragmented methodologies per loan type
  • No borrower segment differentiation
  • Inconsistent interpretation across models
  • No advance warning system (12-month horizon)

Expected Outcome

  • Robust accuracy target of 90%+
  • Structured + unstructured data fusion
  • Analytical methods per loan type and profile
  • Common interpretation framework
  • Consistent, comparable & actionable insights
  • 12-month advance default detection
Solution Design

End-to-End Architecture

Seven-layer pipeline from raw data ingestion to explainable predictions

📥

Data Ingestion

Loads structured (CSV/Parquet) and unstructured text data. Generates synthetic datasets for development. Creates 12-month default labels.

CSVParquetTextExternal
🧹

Preprocessing

Handles missing values, outlier removal (IQR), label encoding, scaling, date decomposition. Text cleaning with NLP pipeline.

ImputationScalingEncodingNLP
🔧

Feature Engineering

Financial ratios, risk indicators, interaction terms, temporal features, loan-type-specific features, and polynomial expansions.

50+ FeaturesRatiosInteractionsAggregations
🤖

Model Training

6 algorithms with SMOTE balancing, stacking ensemble, loan-type-specific models, and hyperparameter tuning via Optuna.

XGBoostLightGBMStackingSMOTE
📈

Evaluation

AUC-ROC, F1, Brier score, calibration analysis, lift charts, threshold optimization, and per-segment breakdowns.

AUC-ROCCalibrationLiftThreshold
🔍

Interpretation

SHAP-based explanations, feature importance, per-segment analysis, and a consistent framework across all loan types.
